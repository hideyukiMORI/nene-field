# ADR Policy

NeNe Field records significant architectural decisions as Architecture Decision Records (ADRs).

## When to write an ADR

- Public PHP API / HTTP endpoint architecture changes
- Dependency package selection
- Frontend framework or tooling decisions
- Storage architecture changes
- Deviations from NENE2 governance
- AI feature policy changes
- Security architecture

## File naming

`docs/adr/NNNN-kebab-case-title.md`

Use the template at `docs/adr/0000-template.md`.

## Status values

- `proposed` — under discussion
- `accepted` — agreed; applies to new work
- `superseded` — replaced by a later ADR
