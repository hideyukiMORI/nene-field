# ADR 0007: AI Summary Policy

## Status

proposed

## Context

NeNe Field's primary differentiator is AI-generated summaries of daily reports.
Managers receive a one-line summary and keyword tags instead of reading full reports,
reducing review burden significantly.

Daily reports contain personal information (PII) including:
- Staff names, work locations, client names
- Incident descriptions that may mention specific individuals
- Health and safety observations

Sending this data to external LLM APIs (OpenAI, Anthropic, etc.) raises
privacy concerns that must be addressed before implementation.

## Decision

**AI summary is opt-in at the organization level.** It is disabled by default.
Admins explicitly enable it in organization settings, confirming they accept
the data processing terms.

**Supported backends (Phase 3):**
1. **External API (default when enabled):** OpenAI-compatible API endpoint.
   Configurable via `NENE_FIELD_AI_API_URL` and `NENE_FIELD_AI_API_KEY`.
   Operators choose their own provider.
2. **Local (future):** Self-hosted Ollama-compatible endpoint for operators who
   cannot send data externally.

**Data minimization:**
- Only the `body` text of the report is sent. Attachments are never sent.
- Metadata (user name, date, project) is **not** sent in the prompt.
- The prompt instructs the model to return only a summary and tags.

**Storage:**
- `reports.ai_summary` (nullable text): one-line summary
- `reports.ai_tags` (nullable JSON array): keyword tags
- Summary is generated once on submission and cached. Re-generation is admin-triggered.

**Audit:**
- `AuditEvent` is recorded when a summary is generated or cleared.

**Transparency:**
- The admin UI indicates "AI summary enabled" per organization.
- The submitter UI shows "summary generated by AI" label on their report detail.

## Consequences

- Organizations that disable AI summary get no PII exposure to external APIs
- Operator must configure API key; it is not bundled
- Local LLM path deferred to Phase 4 (Ecosystem Links milestone)
